### Contributing Guidelines

Thank you for contributing to this project! As a contributer, you must agree to the following guidelines. Any failiure to do so will result in a decline:

Licensing Compliance
All contributions must comply with the project's licensing terms. Please ensure that any code or content you submit does not violate third-party licenses. Please follow GPL 3.0, feel free to ask a maintainer for help.

Code Quality
Ensure your code compiles without errors. Code that does not compile or has issues preventing build or execution will be rejected.
Follow the project's established coding style and formatting guidelines. This helps maintain consistency, and maintainability across the codebase.

Commit Messages
Write clear, concise commit messages that describe the purpose of your changes. Follow the these commit message conventions (e.g., commit messages like feat:, fix:, docs:, etc.).

Respect Existing Issues and Discussions
Before submitting a new feature request or bug report, please check the existing open issues to ensure your request or report hasn't already been addressed. For feature requests, try to provide as much detail as possible, including use cases, type (e.g, ESP, Aimbot, etc.), or any other details.

Be Respectful
Treat others with kindness and respect. Maintain a collaborative, positive environment, and engage constructively with all contributors and maintainers.

We appreciate your contributions and look forward to working together to improve this project!
